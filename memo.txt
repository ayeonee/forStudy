p3.
-HTML : Hyper Text Markup Langage 
-하이퍼 텍스트 : 하이퍼 링크 이용해 독자가 원하는 순서에 따라 기존문서에서 다른 문서로 접근할 수 있는 텍스트.
-마크업 : 태그를 이용하여 문서나 데이터의 구조를 명시하는 언어. 문서나 데이터의 구조를 표현.
-종합하면, HTML은 웹페이지의 구조 혹은 데이터 작성을 위한 마크업 언어. ex.<제목>HTML 배우기<제목>

p4.
-CSS : Cascading Style Sheets 
-스타일 시트 : 웹페이지의 스타일, 디자인을 정의하여 정리한 문서
-캐스캐이딩 : 폭포가 되어 떨어지다. CSS에 적혀있는 스타일을 우선순위에 따라 화면에 적용. 우선순위가 높은 스타일부터 낮은 스타일까지 순차적용.

p.13 HTML의 기본 구성
-컨텐츠(내용)에 태그(마크)를 씌워 구조를 구분하여 정의.
-<시작태그>컨텐츠</종료태그>   --> 이 한덩어리를 요소(element)라고 함.
-제목 : <h1> 부제목 : <h2> 내용 : <p>

p.14
<!DOCTYPE html> //문서형식을 정의할 때 사용. HTML 문서의 가장 처음에 배치되어 이문서가 HTML 문서라고 브라우저에 알려주는 역할 수행.
<html lang="ko"> //<html>태그는 전체 HTML 문서를 감싸는 태그. 문서에서 단 한번 사용 가능. lang="ko"는 이 페이지의 주 언어가 한국어임.
	<head>

	<body>

p.15
<head> : HTML 문서에 대한 정보를 담고 있음, 일반 사용자는 웹 사이트에서 <head> 안에 속한 정보를 볼 수 없음.
	HTML 문서 안에 단 하나 존재, <html>태그 바로 아래 존재. <head>안에 담을 수 있는 태그는
	<meta> : 문서와 관련된 정보를 담는 태그. 브라우저만 읽을 수 있음. 
	<title> : 웹 페이지의 제목을 담는 태그. 브라우저에서 탭의 제목이나 즐겨찾기했을 때 이름으로 볼 수 있음.

<body> : HTML 문서 중 사용자에게 실제로 보여지는 부분. 문서내 단 하나 존재. 
	<h1> : 제목을 표시하는 태그. 1~6까지 중요도. 1이 가장 높음.
	<p> : 단락을 표시하는 태그. 글, 본문에 해당하는 내용 포함.
	이외에도 다양한태그 포함 가능.
<head> 태그 내부와 <body> 태그 내부에 존재하는 태그는 종류에 상관없이 여러번 사용 가능.

p.16 기본적인 레이아웃
시맨틱 태그(Semantic tag) : 의미를 가지는 태그. 나눠진 구획의 기능을 태그 이름만으로도 이해 가능.
	<header> : 웹페이지 혹은 <section>의 소개나 제목을 담기 위해 사용.
	<nav> : 내비게이션 역할을 수행하는 요소. 페이지 이동을 위한 메뉴를 주로 담음.
	<section> : 기준에 따라 구획을 구분하기 위해 사용. 기준에 맞는 <article>들을 담음.
	<article> : 주 내용을 담기 위해 사용.
	<aside> : 광고 또는 사이트의 주변 부분에 해당하는 내용을 담기 위해 사용.
	<footer> : 웹사이트 가장 아래에 들어가는 추가 정보를 담기 위해 사용.

p.19 제목과 본문 태그
제목 태그(Heading tag) : 제목을 나타내고 싶을 때 사용. <h1>~ <h6>, 일반적으로 대제목이 <h1>.
	제목 태그로 감싸는 것만으로도 다른 태그로 감싼 글씨와 다르게 보임. 이미 지정한 CSS가 있기 때문에.
본문태그 <p> : paragraphs의 약자, 단락 혹은 문단이란 뜻. 서로 이어지는 내용의 텍스트를 하나로 묶어 <p>로 감쌈.
본문태그 <br> : break의 약자. 엔터와 동일한 역할, 줄바꿈을 수행. <br> 태그가 적용되지 않는 <p> 태그는 내부에서 엔터를 입력해도 줄바꿈 적용되지 않음.
*빈 요소(Empty element) : <br>처럼 종료 태그를 쓰지 않아도 올바르게 인식하는 태그. 종료 태그가 없어서 컨텐츠가 비어 있고 태그 사이가 비어 있기 때문에 빈 요소라고 함.
본문태그 <pre> : preformatted의 약자. 형식화된 텍스트를 브라우저에 그대로 표시. 

p.20 나누기 위한 태그
<div> : division의 약자, '분할'. 아무것도 나타내지 않고 그저 각각의 태그를 구분짓기위해 사용되는 태그. 아무것도 나타내지 않고 그저 각각의 태그를 구분 짓기 위해 사용.
	굉장히 자주 쓰이며 보통 CSS를 이용하여 레이아웃을 배치할 때 CSS를 효과적으로 적용하기 위해, 또는 묶어서 사용하기 위해 많이 사용.

p.21 글자와 관련된 태그
<strong> : 태그로 감싼 단어 혹은 문장을 볼드체로 바꿈.
<em> : emphasized의 약자, 태그로 감싼 단어 혹은 문장을 이탤릭체로 바꿈.
<sup> : 태그로 감싼 단어 혹은 문장을 일반 위치보다 위로 올림. 
<sub> : 태그로 감싼 단어 혹은 문장을 일반 위치보다 아래로 내림.
<ins> : 태그로 감싼 단어 혹은 문장 아래 밑줄을 추가.(insert)
<del> : 태그로 감싼 단어 혹은 문장에 취소선을 추가.(delete)

p.24 링크 태그란
링크 태그<a> : ahchor(닻)의 약자, HTML에서 하이퍼 링크의 역할을 수행. 이 태그를 사용하면 링크를 통해 다른 웹페이지로 이동하거나 문서로 이동할 수 있는 통로가 마련됨.
	-태그와 컨텐츠, 속성으로 구성되어 있음. ( <a 속성>컨텐츠</a> )
	-속성(attribute)은 태그의 특징, 즉 태그에 대한 추가 정보의 모음. 모든 태그에 속성을 입력할 수 있지만 항상 기재되어야하는 것은 아님. 다만 <a>는 속성을 필수로 가짐.
	-링크 태그의 필수 속성은 주소. 링크 주소라는 종류는 키(Key), 실제 링크 주소는 값(value)으로 생각.
		<a 키="값">구름IDE</a>
	-만약 태그가 여러개의 속성을 갖고 있는 경우 띄어쓰기로 구분. 링크 태그 주소 이외에 이름 속성을 추가하면 <a 주소="http~" 설명="구름IDE로 가는 링크">구름IDE</a>
	
p.25 링크태그 <a>의 속성
href : <a>의 태그는 href를 써야 올바르게 동작. Hypertext Reference의 약자. <a>가 참조하는 웹사이트 주소(경로)를 값으로 가짐.
	<a href="https://ide.goorm.io">구름IDE</a>
target : 링크를 클릭했을 때 해당 페이지를 어디에서 열지 정하는 속성. (링크를 여는 장소는 새 탭, 새 창, 보던 탭 등 다양함)
	사용할 수 있는 대표 값은 _self (현재 탭에서 링크 열기), _blank(새 탭 혹은 새 창에서 링크 열기)
		<a href="https://ide.goorm.io" target="_self">구름IDE</a>
		<a href="https://ide.goorm.io" target="_blank">구름IDE</a>

p.26 웹사이트의 경로와 주소
pah(경로) : 출발지부터 목적지까지 가는 길. 프로그래밍에선 경로를 /로 표시. (ex. https://edu.goorm.io/lecture/한-눈에-끝내는-html5-css3)
URL : 웹사이트 주소(domain) + 경로(path). Uniform Resource Locator
	웹에서 HTML 페이지, CSS문서, 이미지 등을 자원(Resource)라고 부르고, URL은 인터넷에서 자원의 위치를 표현. 
	절대 URL (Absolute URL) : 접근하는 최초 시작점(주소)부터 경유한 경로를 모두 기록. 
	상대 URL (Relative URL) : 현재 위치(기준점)를 기준으로 상대적인 경로를 기록.
	
p.29 이미지 태그
<img> : 넣고자 하는 이미지에 대한 속성을 필히 작성해야 하는 태그. 빈 요소이기 때문에 종료 태그 작성x.
		<img src="이미지 url"/>
	src : source의 약자, 불러올 이미지의 파일 경로 또는 URL을 속성값으로 가짐.
	alt : Alternative text(대체 문구)의 약자, 이미지가 정상 출력되지 않거나 이미지 파일이 아예 존재하지 않는 경우, 이미지 대신 표시할 문구를 값으로 가짐.
		<img src="이미지 url" alt="이미지 대체 문구"/>
	width, height 추가 가능 but 권장x, css에서 웹  페이지와 관련된 모든 스타일을 정의하기에 스타일 관련 부분은 CSS로 조정.

p.30 오디오와 비디오 태그
<audio> : 오디오 파일이 저장된 경로를 src 값으로 설정, 웹페이지에 해당 오디오 파일을 재생하는 플레이어가 추가.
	<audio controls>
		<source src="assets/audio/dance.mp3" type="audio/mpeg">
	</audio>
	확장자의 음원 형식을 뜻하는 type 속성은 생략 가능.
<video> : 비디오 파일이 저장된 경로를 src값으로 설정, 해당 비디오 파일을 재생할 수 있는 플레이어가 웹 페이지에 추가.
	<video controls>
		<source src="assets/video/dance.mp4" type="video/mp4">
	</video>

p.31 유튜브 영상 삽입
<audio>나 <video> 태그는 잘 쓰이지 않음. 
유튜브 영상 아래 공유하기 클릭 > 퍼가기 > 복사 붙여넣기
	<iframe width="560" height="315" src="https://www.youtube.com/embed/gmgcRWxhmqY" frameborder="0" allow="accelerometer; autoplay; encrypted-media; gyroscope; picture-in-picture" allowfullscreen></iframe>
	/embed/뒤는 영어와 숫자를 조합한 youtube영상 ID. 이걸 바꾸면 원하는 영상으로 변경 가능.
	iframe 태그는 유튜브 영상 뿐 아니라 웹 페이지를 삽입할 때도 사용.
